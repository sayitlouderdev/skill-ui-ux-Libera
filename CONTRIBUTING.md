# Contributing

Contributions are welcome — particularly updates to keep the skill current with evolving web standards.

## What to contribute

- **Standard updates** — new browser APIs reaching Baseline, changed support percentages, deprecated patterns
- **Rule additions** — missing UX or accessibility patterns that should be in the checklist
- **Bug fixes** — incorrect CSS, outdated API names, broken script behavior
- **Stack additions** — new stack-specific guidelines for frameworks not yet covered

## What not to contribute

- Changes to the CSV databases from upstream (open issues/PRs on [ui-ux-pro-max](https://github.com/yuvalsuede/ai-component-generator) instead)
- Aesthetic opinions without cross-browser rationale
- Framework-specific rules that don't generalize to the domain

## How to contribute

1. Fork the repository
2. Create a branch: `fix/view-transitions-support` or `add/css-anchor-positioning`
3. Make your changes with a clear commit message
4. Open a pull request describing what changed and why

## CSV databases

The CSV files in `data/` are from ui-ux-pro-max and are not maintained here. If you find errors in the databases, please contribute upstream at https://github.com/yuvalsuede/ai-component-generator.
